Personal Profile

Project Idea

Overview

My project idea is an app that allows the user to learn and memorise lines for monologues, speeches and presentation. What separates this idea from similar apps is that it allows the user to voice recognition to grasp errors they are making in their speech, and as well it allows the user to personalise their learning experience using a variety of different learning methods that best suites the user.

Motivation

Almost every person in the modern day will face the ordeal of presenting a speech in front of an audience. It may be a middle school project in front of a class, business proposition to a board or a monologue in front of a packed-out theatre, the stress of presenting should be made a bit easier with the aid from the modern day guidebook, a phone app. The market for this kind of app is massive, as it effects basically all age and gender demographics. Most similar apps can have easier alternatives such as voice memos for aural learners and simply a notepad or learning cards for those visual learners. There is no other app that uses voice recognition to assist memorisation.

Description

This app is designed to assist users with the task of learning and memorising words for speeches, plays, presentations and any activity that requires the memorisation of words. The main feature of this application that separates it from similar apps is that it uses voice recognition to aid you in the memorisation of a text. Specifically, the user will insert the text they want to memorise, once inserted, they record themselves without the original text as a prompt, then this recording is then converted into text and compared with the speech inserted, depicting to the user mistakes. These mistakes can be missing lines, incorrect words and words that were not articulated properly or were mumbled. The results can be given as percentile value based on how accurate the user was compared to the original text they inserted and with errors clearly highlighted.

Furthermore, before the user uses the voice recognition portion of learning they can participate in many other learning styles for memorisation. Such as recording themselves performing a text and then listening to it back for those who prefer to learn through aural methods. User can also learn visually, once the text has been inputted into the application, the first line of the text will appear and once the user has read it the text disappears, the user then can speak the line and then they will be prompted if the spoke the line correctly. If they are correct another line will then be added, they will read both lines and the process will continue. But if they are wrong the must repeat the lines until they are correct. Users can personalise these options, for example if they are doing a Shakespeare monologue in which the lines have to be 100% correct, it will allow the user not to go to the next line unless they are completely correct. But if they are doing a presentation in which the lines can accuracy is not of up most importance, it will allow the user to continue to the next line. As well, the app can take into account of synonyms of words. For example if the line is “The book was good” and the user says “The book was great”, this will not be considered a mistake if requested by the user.

The program will allow multiple user work together from 1 device. If the users a learning lines for a play it will allow the lines to be split amongst the characters. Then the users can go over the lines together using the voice recognition feature, it will tell the users which lines are correct and any errors made.

The app will collate data on previous attempts at learning the text, it will the convey to the user any mistake patterns that are consistently occurring in their learning process. That may be a certain line they keep forgetting or a word that they consistently change.

The app has the potential to be a database that allows users to upload video or audio files of them performing their text. It can allow feedback from other users and allow users to compare their own performances. If the user is learning a Shakespeare monologue, they have access to other users’ interpretations of the monologue. It creates a communal environment where people can learn from each other, not just by themselves.

Tools and Technologies

There are many open-source voice recognition software available to the public such as Project Deepspeech, Kaldi and Leon. Many of these program’s API supports code such as C, Python and Java. The most likely software to be chosen is Deepspeech as it can support multiple platforms for its development, some include Windows, Linus and MacOS. A coding language has to be chosen for development for the app, the code the will most likely be chosen is python since that is what is being taught in the Introduction to Programming course in the Bachelor of Information Technology degree at RMIT. An app development software could be used to minimise coding for creating the personalisation and template of the app. Software include Appery.io and Appy Pie.

Skills Required

Skills that are required to develop this app, is a proficiency in coding for app development. These skills can be gained via online platforms such as Scrimba and Code Academy that aim to teach people how to code through a multitude of tasks and classes. As well as a understanding of the process of voice recognition and implementing the open source code into an application. Furthermore, the ability to analyse data collated on the topic of memorisation to implement the best learning solutions for specific users.

Outcome

If this project is successful it will create a resource where people can effectively learn and memorise a particular text that they must present to an audience. It will allow the user to not reinforce bad habits such as consistently forgetting a line or replacing particular words as the app compares results from previous attempts. Overall, this app can really help people who struggle with learning speeches, monologues and any piece that requires memorisation.
